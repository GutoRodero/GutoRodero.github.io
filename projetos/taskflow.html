O problema

Times pequenos precisam de uma ferramenta simples para organizar o trabalho sem a complexidade (e o custo) das grandes plataformas como Jira ou ClickUp. Ferramentas comerciais são caras, lentas e cheias de features que ninguém usa.

A solução

TaskFlow é um Kanban clean, com foco no essencial: cartões arrastáveis, colaboração em tempo real via WebSockets, comentários e menções. A API segue padrão REST + eventos em socket, e o frontend usa React com estado global via Zustand.

Arquitetura

O backend expõe uma API REST para operações CRUD e usa Socket.io para eventos em tempo real (novo cartão, movimentação entre colunas, comentários). O banco é PostgreSQL com schema normalizado, e as senhas são hashed com bcrypt. O frontend consome os endpoints via fetch + React Query para cache inteligente.

O que aprendi

Este projeto me forçou a explorar melhor arquitetura full stack, autenticação segura e o desafio de sincronização em tempo real com múltiplos clientes conectados. WebSockets parecem simples até você tentar lidar com reconexão, mensagens duplicadas e ordem de eventos.
